## 1. 결정 로그 (왜 이렇게 만들었나)

처음 아이디어는 위키를 로컬에 MD로 저장하는 것. 논의하며 방향을 좁혔다.

| 선택지 | 방식 | 결론 |
|---|---|---|
| **A. 구조화 ZIP 내보내기** | 모든 위키를 구조화해 `.zip` 다운로드 | ✅ **채택(중심)** — 전 브라우저 동작, 손실·충돌 위험 없음 |
| B. 로컬 폴더 동기화 | File System Access로 폴더에 자동 쓰기 | ⚠️ 옵트인으로만 — 데스크톱 Chrome 한정, 권한 재허용 |
| C. 오프라인 캐시 | IndexedDB 보관 | 제외 — "로컬 파일"이 아님 |

**핵심 통찰**
- 목적이 "옵시디언/에이전트에 먹이기"라면 본질은 *저장 방식*이 아니라 **출력 포맷의 구조**다. → 볼트 생성기 하나를 잘 만들고, ZIP/폴더는 그 위의 "배달구".
- 상시 자동 동기화(B)의 트레이드오프: ① 데스크톱 Chrome 전용 ② 매 세션 권한 재허용(무인 불가, PWA 영구권한으로 완화) ③ **단방향이면 옵시디언 쪽 수정이 덮어써져 소실** — 그래서 자동 동기화는 중심에 두지 않고 옵트인으로.
- "정확한 백업"이면 MD보다 JSON이 무손실. → 둘 다 제공.

## 2. 구현 범위 (요청 항목 전부)

기반(구조화 볼트) + 후속 1·2·3·4·5·6·7을 모두 반영. 모든 처리는 **클라이언트 사이드·읽기 전용**(가져오기만 신규 문서 생성).

### 기반 — 구조화 볼트 내보내기
- `ExportDialog` 스코프: **이 페이지 / 하위 포함 / 전체 볼트 / 선택**. 다중 스코프 → `.zip`, 단일 페이지 → `.md`.
- `buildWikiVault(opts)`: 위키 트리를 폴더 계층으로 미러링(자식 있는 노드 = `폴더/폴더.md` folder-note), 노트별 **YAML frontmatter**, `index.md`(MOC) + `manifest.json` + `README.md`.
- 공용 `wikiBlocksToMarkdown()` 추출 + **기존 단건 변환 버그 수정**(`code.language`·`math.content`·`image.caption` 필드 오류, `attach` 누락), 엔티티 디코드·인라인 서식 보존·콜아웃 → 옵시디언 콜아웃 매핑.
- 파일명 슬러그(FS·위키링크 깨짐 문자 제거, 유니코드 유지) + 형제 동명 디듀프 + `[[wikilink]]`가 실제 파일명을 가리키도록 정합.

### 후속 항목
| # | 기능 | 요지 |
|---|---|---|
| **1** | JSON 사이드카 | `data/pages.json`에 원본 블록(무손실) — 정확 백업·재가져오기 기반 |
| **2** | 단일 번들 | `bundle.md`로 전체 이어붙임 — 일부 AI 단일 입력에 편리 |
| **4** | 이미지 로컬화(M2) | `assets/`로 받아 노트 깊이별 상대경로 치환, 실패 시 URL 유지 |
| **5** | 선택 내보내기 | 체크박스 페이지 트리, 상위는 경로 유지 위해 자동 포함 |
| **7** | 폴더 직접 쓰기 | File System Access 옵트인 + IndexedDB 핸들 보존("지난 폴더에 저장") |
| **3** | i18n | 신규 문구 en/ja/zh/es + 기존 phrase 파일 파싱 버그 수정 |
| **6** | 가져오기 | `.zip`/`pages.json`/`.md` → 새 페이지, 부모-우선·id 재매핑으로 계층 보존 |

## 3. 볼트 출력 구조

```
plannary-vault-YYYYMMDD/
  index.md            # 전체 트리 목차(MOC) · 에이전트 진입점
  manifest.json       # 기계 순회용 트리·경로·태그
  README.md           # 볼트 설명 + 생성 시각
  bundle.md           # (옵션) 전체를 한 파일로
  data/
    pages.json        # (옵션) 무손실 원본 블록
  notes/
    핸드북/
      핸드북.md         # 자식 있는 노드 = 폴더 + 동명 .md
      디자인 시스템/
        디자인 시스템.md
        컬러 토큰.md
  assets/
    img-<hash>.png    # (옵션) 이미지 로컬 사본
```

**노트 파일 = frontmatter + 본문**
```markdown
---
id: w7
title: "회의록"
parent: w1
up: "[[프로젝트]]"
tags: ["기획", "2026"]
icon: "📘"
created: 2026-05-01
updated: 2026-06-20
source: plannary
---
# 회의록

…정제된 마크다운 본문…
```

## 4. 사용 방법 (UI)

위키 페이지 헤더의 **`···` 메뉴**:
- **내보내기** → 스코프 선택 → (볼트면) 포함 항목 체크(이미지/JSON/번들) → **ZIP 다운로드** 또는 **폴더에 저장**(지원 환경).
- **가져오기** → `.zip`·`pages.json`·`.md` 선택 → 새 페이지로 추가(기존 문서는 불변).

## 부록 — "학생 편의 자동화" 아이디어에 대한 메모

별도로 제안된 *강의 자동재생 출석·안전교육 자동 이수* 아이디어는 **거짓 출결/거짓 인증**에 해당(장소가 Plannary든 별도 앱이든 동일, ToS·안전·법적 책임 리스크)하여 자동 완료 형태로는 진행하지 않기로 함.

대신 같은 불편(까먹어서 놓침)을 합법적으로 푸는 방향을 권장:
1. **연례 의무 트래커** — 안전교육 등 매년 반복 의무를 이수일 기준 1년 뒤 자동 재생성 + 알림
2. **온라인 강의 시청 마감(watch window) 알림** — 마감 전 리마인드(영상은 본인이 시청)
3. e-Class 연동으로 의무 항목·마감 surface, 진행률/체크리스트

→ "자동 완료"가 아니라 **자동 추적·알림**으로 학생 편의를 살리는 게 핵심.
