# 🤖 GEMINI_RULES.md: 프로젝트 전용 AI 에이전트 가이드라인

이 파일은 이 프로젝트를 수정하는 모든 AI 에이전트가 반드시 준수해야 할 **최상위 규칙**을 정의합니다.

## 1. 🏗 아키텍처 원칙 (SPA 무결성)
- **일방통행 라우팅**: `window.location.href`나 `<a>` 태그의 기본 이동을 사용하지 않습니다. 모든 페이지 전환은 `app.js`의 `switchPage(targetId)` 함수를 통해서만 수행해야 합니다.
- **섹션 기반 구조**: 새로운 페이지를 추가할 때는 `index.html` 내에 `<section id="page-..." class="page-content">`를 생성하고, 내비게이션 요소에 `data-target` 속성을 부여합니다.

## 2. 🎨 UI/UX 스타일 가이드
- **글래스모피즘 계승**: 모든 새로운 컴포넌트는 기존의 투명도(Backdrop Filter), 블러, 둥근 모서리(`border-radius: 16px`) 스타일을 유지하여 일관성을 갖춰야 합니다.
- **CSS 변수 활용**: 색상이나 폰트 크기 변경 시 `style.css` 상단의 로컬 변수를 우선적으로 참조하여 사용합니다.
- **인라인 스타일 금지**: HTML 태그에 직접 `style="..."`을 작성하지 말고, 명확한 클래스명을 부여하여 `style.css`에서 관리합니다.

## 3. 💾 데이터 및 보안 규칙
- **실시간 바인딩**: `db.collection(...).onSnapshot`을 통해 데이터가 즉시 반영되도록 구현합니다.
- **파괴적 작업 방어**: 삭제(`delete`) 버튼 클릭 이벤트에는 반드시 `confirm('...')` 다이얼로그를 추가하여 사용자 실수를 방지합니다.
- **UID 귀속**: 모든 `add`나 `set` 요청 시 `currentUser.uid` 필드를 정확히 포함하여 데이터 소유권을 유지합니다.

## 4. 📝 코드 문서화 및 커뮤니케이션
- **한글 주석**: 모든 함수와 비즈니스 로직에 대한 설명 주석은 **한국어**로 작성합니다.
- **친절한 피드백**: 코드 수정 후 사용자에게 `showToast()`를 통해 작업 결과를 시각적으로 알려야 합니다.
- **커밋 메시지**: Git 커밋 시 변경 사항을 명확하고 짧게 설명합니다.

---
**주의**: 이 규칙을 어길 시 사용자 경험(UX)이 손상될 수 있으며, 프로젝트의 유지보수가 어려워질 수 있습니다.
